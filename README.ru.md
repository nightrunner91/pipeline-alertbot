## Обзор проекта

**Pipeline Alertbot** -- это лёгкий, готовый к продакшену ретранслятор вебхуков, который доставляет события GitLab CI/CD пайплайнов в групповые чаты Telegram. Бот принимает вебхуки от GitLab, форматирует их в читаемые уведомления и отправляет в указанный Telegram-чат в реальном времени.

Бот предназначен для команд, которые хотят мгновенно получать информацию о состоянии CI/CD пайплайнов, не покидая Telegram. Поддерживает несколько стилей уведомлений, безопасную валидацию вебхуков и легко развёртывается на платформах вроде Railway.

## Возможности

- **Уведомления в реальном времени** -- Мгновенные оповещения о запуске, успешном завершении, ошибке и отмене пайплайнов
- **Три стиля уведомлений** -- Выбирайте между форматами card, badge или minimal через переменную `ALERT_STYLE`
- **Интерактивные кнопки** -- Прямые ссылки на пайплайн, коммит и репозиторий прямо в Telegram
- **Безопасная валидация вебхуков** -- Проверка входящих запросов по секретному токену GitLab
- **Привязка к нескольким портам** -- Автоматическая привязка к нескольким портам для совместимости с динамической маршрутизацией Railway
- **Надёжная обработка ошибок** -- Глобальные обработчики необработанных исключений и отклонённых промисов предотвращают тихие аварийные завершения
- **Эндпоинт диагностики** -- Эндпоинт проверки работоспособности, сообщающий статус конфигурации без раскрытия секретов
- **Безопасный запуск** -- Бот инициализируется независимо от вебхук-сервера; отсутствие учётных данных не блокирует запуск сервера

## Начало работы

### Требования

- Node.js >= 18
- npm (поставляется вместе с Node.js)
- Токен Telegram-бота от [@BotFather](https://t.me/botfather)
- ID Telegram-группы или канала для отправки уведомлений
- Доступ к настройкам вебхуков в проекте GitLab

### Установка

1. Клонируйте репозиторий:

   ```bash
   git clone https://github.com/your-org/pipeline-alertbot.git
   cd pipeline-alertbot
   ```

2. Установите зависимости:

   ```bash
   npm install
   ```

3. Создайте файл `.env` на основе шаблона:

   ```bash
   cp .env.example .env   # или создайте .env вручную
   ```

### Быстрый старт

1. Заполните `.env` необходимыми учётными данными (см. [Конфигурация](#конфигурация)).
2. Запустите приложение:

   ```bash
   npm start
   ```

3. Настройте вебхук GitLab, указав URL вашего сервера (см. [Настройка вебхука GitLab](#настройка-вебхука-gitlab)).
4. Запустите пайплайн в GitLab и наблюдайте за появлением уведомления в Telegram.

## Конфигурация

### Переменные окружения

Создайте файл `.env` в корне проекта со следующими переменными:

| Переменная | Обязательная | По умолчанию | Описание |
|---|---|---|---|
| `TELEGRAM_BOT_TOKEN` | Да | -- | Токен бота, полученный от [@BotFather](https://t.me/botfather) |
| `TELEGRAM_GROUP_ID` | Да | -- | ID целевого чата (например, `-1001234567890` для групп) |
| `GITLAB_WEBHOOK_SECRET` | Да | -- | Секретный токен для валидации входящих вебхуков GitLab |
| `ALERT_STYLE` | Нет | `card` | Формат сообщения: `card`, `badge` или `minimal` |
| `PORT` | Нет | `3000` | Основной порт вебхук-сервера |

> [!IMPORTANT]
> Никогда не коммитьте файл `.env`. Он указан в `.gitignore` и должен существовать только локально или в хранилище секретов вашей платформы развёртывания.

### Стили уведомлений

Переменная `ALERT_STYLE` управляет тем, как уведомления о пайплайнах выглядят в Telegram:

| Стиль | Описание | Пример вывода |
|---|---|---|
| `card` | Чистая карточка с жирными заголовками и моноширинными значениями | Заголовок статуса, имя проекта, ветка, коммит, автор, длительность |
| `badge` | Структурированный список с древовидными соединителями и ссылкой "View in GitLab" | Заголовок бейджа, поля в древовидном формате, сообщение коммита, встроенная ссылка |
| `minimal` | Компактный однострочный формат с встроенными метаданными | Статус + проект, встроенные ветка/коммит/автор, ссылка "Open Pipeline" |

Все стили включают интерактивную клавиатуру с кнопками-ссылками на пайплайн, коммит и репозиторий.

### Настройка вебхука GitLab

1. Откройте ваш проект GitLab и перейдите в **Settings > Webhooks**.
2. Установите **URL** на ваш развёрнутый эндпоинт:

   ```
   https://your-domain.com/api/webhook/gitlab
   ```

3. Установите **Secret token** в значение, совпадающее с `GITLAB_WEBHOOK_SECRET` из вашего `.env` файла.
4. В разделе **Trigger** отметьте **Pipeline events**.
5. Оставьте **Enable SSL verification** включённым для продакшен-развёртываний.
6. Нажмите **Add webhook** и протестируйте через выпадающее меню **Test > Pipeline events**.

> [!NOTE]
> Бот обрабатывает только события пайплайнов со статусами: `running`, `success`, `failed` и `canceled`. Остальные события подтверждаются, но игнорируются.

### Развёртывание на Railway

Проект включает конфигурацию `railway.json` для развёртывания в один клик на [Railway](https://railway.com/):

```json
{
  "$schema": "https://railway.com/railway.schema.json",
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "node src/index.js"
  }
}
```

Для развёртывания:

1. Отправьте репозиторий на GitHub.
2. Подключите репозиторий в панели управления Railway.
3. Добавьте необходимые переменные окружения в редакторе переменных Railway.
4. Railway автоматически соберёт и развернёт приложение с помощью Nixpacks.

Сервер привязывается к нескольким портам (`3000`, `8080`, `5000`, `8000`, `4000`, `80`, `5952`) для обработки динамического назначения портов Railway.

## Использование

### Локальный запуск

```bash
npm start
```

Ожидаемый вывод:

```json
{"level":"INFO","timestamp":"2026-05-22T12:00:00.000Z","message":"Telegram bot initialized successfully","context":{}}
{"level":"INFO","timestamp":"2026-05-22T12:00:00.000Z","message":"Starting webhook server...","context":{}}
{"level":"INFO","timestamp":"2026-05-22T12:00:00.000Z","message":"Webhook server listening on 0.0.0.0:3000","context":{}}
```

### Эндпоинт вебхука

- **URL**: `POST /api/webhook/gitlab`
- **Заголовки**: `X-Gitlab-Token: <ваш-секрет>`
- **Тело**: Полезная нагрузка события пайплайна GitLab (JSON)
- **Ответ**: `200 OK` при успехе, `401 Unauthorized` если секретный токен не совпадает, `500` если бот или ID группы не указаны

### Эндпоинт диагностики

- **URL**: `GET /`
- **Ответ**: JSON-объект со статусом сервера и наличием конфигурации (секреты не раскрываются)

```json
{
  "status": "running",
  "env": {
    "PORT": 3000,
    "HAS_TELEGRAM_BOT_TOKEN": true,
    "HAS_TELEGRAM_GROUP_ID": true,
    "HAS_GITLAB_WEBHOOK_SECRET": true,
    "ALERT_STYLE": "card"
  },
  "botInitialized": true
}
```

## Тестирование

Запуск полного набора тестов:

```bash
npm test
```

Выполняются три набора тестов последовательно:

| Скрипт | Описание |
|---|---|
| `npm run test:unit` | Модульные тесты форматирования сообщений, парсинга длительности и форматирования этапов |
| `npm run test:webhooks` | Интеграционные тесты вебхуков с примерами полезных нагрузок |
| `npm run test:security` | Тесты валидации безопасности (проверка токена, санитизация входных данных) |

Тесты используют файлы фикстур из `test/fixtures/`, которые имитируют реальные полезные нагрузки событий пайплайнов GitLab.

## Разработка

### Доступные скрипты

| Команда | Описание |
|---|---|
| `npm start` | Запуск приложения |
| `npm test` | Запуск всех тестов |
| `npm run test:unit` | Запуск только модульных тестов |
| `npm run test:webhooks` | Запуск только тестов вебхуков |
| `npm run test:security` | Запуск только тестов безопасности |

### Добавление нового стиля уведомлений

1. Откройте `src/services/message-builder.js`.
2. Создайте новую функцию форматирования (например, `formatCompactStyle(data)`).
3. Добавьте case в оператор switch функции `buildMessage()`.
4. Добавьте соответствующие тесты в `test/test-unit.js`, используя существующую или новую фикстуру.
5. Запустите `npm test` для проверки.

### Логирование

Приложение использует структурированное JSON-логирование через `src/utils/logger.js`. Все записи журнала включают `level`, `timestamp`, `message` и опциональный `context`. Ошибки дополнительно включают поля `error` и `stack`. Логи записываются в `stdout` (INFO) и `stderr` (ERROR) для удобной интеграции с системами агрегации логов.
